{% comment %} Project override of Unfold's sidebar/app-list badge. Upstream renders the badge box whenever the `badge` KEY is present — for our callback badges (dotted import strings, always truthy) that means a badge would show even when the count is 0, printing a noisy "0". We render ONLY when the callback returned a truthy value, so live "needs-action" counts appear and idle queues stay clean. Variant/style classes mirror upstream. NOTE: every badge in this project MUST be a callback (see apps.core.admin_badges). A static, callback-less `badge` value would be suppressed here by design. {% endcomment %} {% if item.badge_callback %} {{ item.badge_callback }} {% endif %}
